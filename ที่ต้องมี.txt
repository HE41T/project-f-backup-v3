ลักษณะพื้นฐานของโครงงานแอปพลิเคชั่นหรือเว็บของนักศึกษา ขั้นพื้นฐานทีต่ ้องมี
โครงงานแอปพลิเคชันและเว็บทีต่ ้องทำเป็นพื้นฐานเสมอคือ
1. มีระบบ multiuser แยกเป็น superuser admin และ user
2. การลบหรือแก้ไขงานต้องมีการยืนยันก่อนจะทำเสมอ
3. แอดมินสูงสุด(superuser)มีคนเดียวมีอำนาจสูงสุดเพิ่มลบแก้ไขได้ทุกอย่าง แต่จะมีแอดมินสำรองทำงานแทนได้ แต่แอดมินสำรองจะลบแอดมินสูงสุดไม่ได้
ให้สิทธิ์แอดมินสูงสุดผู้เดียว
4. การค้นหาข้อมูลต้องมีเสมอ ค้นเป็นช่วงเวลาได้ ควรต้องมีค้นข้อมูลทั่วไปตามลักษณะงานที่ต้องมีอยู่แล้ว
5. จัดเก็บ log การเข้าใช้งานของ user แต่ละระดับ เพื่อให้รู้ว่าเข้าใช้งานและออกจากการใช้งานเวลาไหนเป็นอย่างต่ำ ถ้าทำถึงระดับเก็บ log การ
ใช้งานแต่ละตารางข้อมูลได้ยิ่งดี(ถ้ามีได้ก็ดี)
7. การกรอกข้อมูลต้องให้ตรงรูปแบบชนิดของข้อมูลนั้นๆ เช่น กรอกอีเมล์เบอร์โทรต้องมีรูปแบบตามมาตรฐาน ชื่อนามสกุลไม่มอี ักขระพิเศษไม่เกิน
จำน

อยากได้แบบapi sharpen ที่เป็นส่วนเสริมหลังจากการ resize แล้ว คือจะใช้ระบบ api sharpen ว่าผู้ใช้จะใช้หรือไม่ก็ได้ แบบไม่บังคับ เป็นแค่ส่วนเสริมหลังจาก resize แล้วเท่านั้น แต่ถ้ามีการใช้ ก็ให้ เอารูปล่าสุดมาทำการ sharpen แล้วส่งภาพกลับไปโชว์ที่หน้า frontend

รวมให้อยู่ใน ไฟล์เดียวไม่ได้หรอ sharpen.py
